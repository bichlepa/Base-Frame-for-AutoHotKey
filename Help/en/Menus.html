Menus

This page explains you every option you can make.

Main window


Sourcecode

This field shows you the current main file of the source code. You may start a new project by clicking on the button Open.

Application name

You can edit the application name here. It can consist of any characters. The name of the application will be shown in user dialogs of installer and update checker.

Installation folder

This is the default name of the installation folder. The user can change the folder name during installation.

Start menu folder

This is the default name of the start menu folder. A start menu folder will only be created during a normal installation. It has no effect, if a portable installation is performed.

Application ID

The application ID is used during a normal installation for creating registry entries. It is important that it remains equal for all versions, otherwise the installer will not find an existing installation.

Note! Don't change it if the installer should be able to update an existing application!

Author

The name of the author will be displayed during installation by default.

Email

The email will be displayed during installation by default.

Application version

This value will be displayed on some user dialogs during install. It can be any text. For example: 2.4 BETA

Update version

This version number will be displayed during installation and it is used by the update checker in order to find out whether a new version is available. It must consist only of numbers or dots. For example: 2.4.01

Advanced

Click here in order to find more options.

Included files

This list shows all files and folders which will be included and extracted by the installer.

Compiled Scripts:
All ahk files listed in section Compiled Scripts will be compiled and then included to the installer. You can choose any icon to every compiled script.

Generate installation file

When you click on this button, the generation process will start.

If this button is deactivated, it may have following reasons:
- One of following fields are empty: Sourcecode, Application name, Application ID, Application version, Update version
- The Update checker is enabled, but no link for getting the update informations is set.

Keep temporary files

If this checkbox is checked, the temporary files will be kept. You will find them in the sourcecode directory in the folder named Base Frame temp.


Installer options



Installer language

You can set here a language for the installer. If the desired language is not available, you can contribute to Base Frame by translating it.

Installation type

You can choose, whether a normal and / or a portable installation is allowed.


A normal installation will put the files to the programs folder, optionally make some start menu entries and make some entries in the registry.

A portable installation will extract the files to any folder. It will allow to use the application on an USB drive. Make sure that you don’t enable portable installations for programs that rely on the registry.



Update checker options


Include update checker

If this checkbox is checked, an update checker will be appended to your application.

Behavior

Update checker can automatically check for updates on a startup of the main application. To achieve this, the main script is modified prior to compilation.
If you want to allow the user to choose whether the update checker should automatically check for updates, you may define an entry in an ini file, which will be read on every start of the main application. Depending on the value in the ini file, it will or will not check for updates.

Download URL for update information

If update checker is enabled, at least one update URL must be set. The URL must directly lead to a text file in ini format.
Base frame can create the ini file for you. You must upload it and make it accessible by anyone.

More than one URL may be specified. This may be useful, if the first URL will become corrupt. The update checker will still be able to find an update using the other URLs.

Ini file with update information

Base frame can create the ini file, which will contain update information for the update checker. To enable this, you must check Create ini file after generation and fill out Destination path for ini file. The ini will be created during the generation process.

You may optionally fill out the fields Installer URL and Project website.
Installer URL must be set if the update checker should be able to download and run the Update. The URL must lead directly to the installer.
If Project website is set, it will be opened by the update checker, if no Installer URL is given or the download fails.

License options


Include following license

If this field is selected, you may choose one of the default licenses.

Use own license text

If this field is selected, you may use an own license text. You can add some placeholders inside, which will be replaced:
%AppName% %AppVersion% %AppUpdateVersion% %AppAuthorName% %AppAuthorEmail%


Changelog options


Changelog

You can select a file, which contains the changelog or create a new one by clicking on the button with three dots.

Afterwards you can edit the changelog in the bit edit field. To add an entry you can press the button Add entry. The changelog file will be updated after a click on the button OK.

If a changelog file was specified, it will be shown during the installation. In a future release the update checker will show the changes since the current version.
